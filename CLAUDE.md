# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## О проекте

Семейный финансовый менеджер (Vue 3 + Vite, статическая сборка без бэкенда). Считает
денежный поток по дням, обслуживает много кредитных карт с разными датами выписок и
льготными периодами, кредиты, доходы/расходы с гибкой регулярностью, финансовые цели с
ETA-прогнозом. Мультивалюта (RUB базовая, AMD, USD). Данные живут в localStorage,
переносятся через экспорт/импорт JSON. Интерфейс полностью на русском.

## Команды

```bash
npm install
npm run dev        # dev-сервер, http://localhost:5173
npm run build      # статика в dist/
npm run preview    # предпросмотр собранной статики
npm test           # тесты движка (node --test, чистый Node, без раннера)
```

Три тест-файла: `test/finance.test.js` (ядро дат/прогноза/целей/карт), `test/scenarios.test.js`
(движок сценариев и переносы), `test/advice.test.js` (оптимизатор-советник). Запуск отдельного
теста: `node --test --test-name-pattern 'monthlyFactor'`. Тесты покрывают только чистые функции
из `src/finance.js`, `src/scenarios.js`, `src/advice.js` — Vue-компоненты не тестируются
(проверяются вручную/через Playwright).

Деплой: пуш в `main` собирает и публикует на GitHub Pages (`.github/workflows/pages.yml`).
`vite.config.js` использует `base: './'` (относительные пути ассетов), чтобы одинаково
работало и на Pages в подкаталоге, и при локальном preview.

## Архитектура

Слои: **чистое финансовое ядро** (тестируемое) → **слой сценариев/советника** (тоже чистый,
поверх ядра) → **реактивный стор** (localStorage) → **Vue-компоненты по вкладкам**.

Направление зависимостей строго одностороннее: `advice.js` → `scenarios.js` → `finance.js` →
`money.js`. Обратных импортов нет (например, finance.js НЕ импортирует scenarios.js) — это
осознанно, чтобы не было циклов. Новую логику клади на правильный слой: базовый расчёт денег
и прогноза — в finance.js; ходы/what-if/переносы — в scenarios.js; правила-рекомендации — в
advice.js.

### Ядро (чистые функции, без Vue)

- `src/money.js` — мультивалюта. Каждая денежная величина хранится как
  `{ amount, currency }` в **нативной** валюте записи. `rates` = сколько единиц валюты в
  1 рубле (`amdPerRub`, `usdPerRub`). Всё сводится к рублю через `moneyToRub()` перед
  арифметикой. Никогда не складывай суммы в разных валютах напрямую — сначала в рубли.
- `src/finance.js` — даты, регулярность, движок прогноза, цели, вся математика карт. **Даты
  локальные без сдвига TZ** (`parseDate`/`fmtISO`/`addMonths`/`addDays` работают через
  `new Date(y, m, d)`, а не через UTC-парсинг ISO-строк — это осознанно, не «чини»; никакого
  `toISOString` в продакшен-коде).
- `src/scenarios.js` — движок сценариев (what-if) поверх ядра. `applyScenario` превращает
  «ходы» в форк состояния, `evaluateScenario` прогоняет его через `buildForecast` и считает
  метрики. Расчёт займов с карты и переносов долга. Импортирует finance.js/money.js.
- `src/advice.js` — оптимизатор-советник (правила-эвристики). `cardAdvice` даёт ранжированные
  рекомендации по картам. Импортирует finance.js/scenarios.js/money.js.

### Ключевые концепции движка

- **Schedule** (`{ frequency, interval, customUnit, startDate, endDate }`) — регулярность
  записи. `expandSchedule()` разворачивает её в конкретные даты внутри диапазона;
  `monthlyFactor()` даёт множитель «сколько раз в месяц» для месячных эквивалентов.
  Частоты: `once | weekly | biweekly | monthly | quarterly | yearly | custom`.
- **buildForecast(state, opts)** — главный движок. Собирает события (доходы +, расходы −,
  кредиты −, карты −), сортирует по дате, считает нарастающий остаток от `startingCash`,
  формирует `alerts` при падении ниже `safetyBuffer`. Возвращает `{ days, events, alerts,
  minBalance, endBalance, ... }`. Кредитки: `payStrategy: 'full'` даёт **одно** событие на
  весь долг в грейс (без процентов); `'minimum'` разворачивается в **ряд** ежемесячных
  платежей до закрытия долга (или конца горизонта) через `cardPaymentSchedule` — минимальный
  платёж + проценты на остаток каждый месяц.
- **Модель карты (важно).** Даты цикла хранятся ЯВНО: `statementDate` / `dueDate` /
  `graceEndDate` (ISO-строки), плюс `statementCycleDays`. `cardCycle(card, from)` возвращает
  актуальный цикл на дату `from`, автопродляя сохранённый цикл вперёд помесячно. Старое
  «число месяца» (`statementDay`/`dueDay`) больше не используется — `migrateCard` в store.js
  синтезирует явные даты из старых данных. Долг берётся через `cardDebt(card, rates)` =
  сумма выписки если >0, иначе текущий долг (НЕ пиши `statementBalance || currentDebt` —
  пустой объект `{amount:0}` truthy и дал бы долг 0; это была причина бага «карты не в
  прогнозе»). Минимальный платёж: `cardMinCore` (тело: % от остатка или фикс, без процентов)
  + проценты если `minPaymentPlusInterest`; `cardMinPayment` = обёртка от текущего долга.
- **Переводы между картами.** Поля: `transferGraceEnabled` (действует ли грейс на перевод —
  у Т-Банка true, у Озон/Уралсиб false), `transferLimit` (беспроцентный лимит),
  `transferGraceDays`, `transferFeePercent`/`transferFeeFixed` (комиссия). `transferCost` в
  scenarios.js считает цену переноса; `availableLimit = min(transferLimit, свободный лимит)`.
- **Сценарии (scenarios.js).** Ход (`move`): `purchase` (крупная покупка), `cardLoan` (заём с
  карты), `newLoan` (новый кредит), `adjust` (разовый доход/расход), `transfer` (перенос
  долга С карты `fromCardId` НА `toCardId`). Модель займа/переноса — «модель A»: рост долга
  карты-плательщика в applyScenario НЕ отражается, возврат моделируется парным событием в
  evaluateScenario (иначе двойной счёт). Проценты — простой формулой `apr × сумма × дни/365`.
- **cardsSummary(state, opts)** (finance.js) — агрегаты по картам для вкладки «Стратегия»:
  проценты за горизонт, месячная нагрузка, долг (грейс/под %), свободный лимит, `perCard`.
- **cardAdvice(state, opts)** (advice.js) — правила-эвристики: critical (кассовый разрыв к
  платежу карты), save (перенос дорогого долга под грейс с расчётом экономии; порядок
  погашения). Каждая рекомендация с обоснованием и опциональным ходом-переливом.
- **buildMonthly()** — усреднённая картина «в среднем за месяц» (доход/расход/обязательства/
  профицит), используется дашбордом и целями.
- **computeGoals(state, opts)** — ETA по целям. Свободный месячный профицит распределяется
  **последовательно по приоритету** на цели без своего взноса; цели с ручным
  `monthlyContribution` считаются параллельно. `opts.extraIncome/extraExpense` — режим
  «играем цифрами» (what-if).

### Стор

`src/store.js` — `reactive(load())`, автосохранение в localStorage (debounce 200 мс,
ключ `family-finance:v1`). `migrate()` добивает отсутствующие поля при загрузке старых
данных, `migrateCard()` — отдельно поля карты (даты цикла из старых чисел, дефолты полей
переводов); при добавлении новых обязательных полей в модель обновляй их. `state.scenarios` —
массив сценариев (тоже персистится). CRUD через `addItem/removeItem/duplicateItem(kind, ...)`,
где `kind` ∈ `income | expense | loan | card | goal` (сценарии ведутся прямым доступом к
`state.scenarios`). `src/seed.js` (`makeSeed()`) — стартовый префилл;
поле `note` на записях — подсказка-плейсхолдер для пользователя, не логика.

### UI

`src/App.vue` — вкладки (dashboard/forecast/goals/scenarios/cards/cardStrategy/loans/income/
expenses/settings), переключение через `<component :is>`. Активная вкладка — реактив `nav`
в store.js (`nav.active` + `goTab(key)`), чтобы компоненты могли переключать вкладку
(например кнопка «Проиграть в Сценариях» на вкладке «Стратегия»). Каждая вкладка — компонент
в `src/components/` (Сценарии — `ScenariosView.vue`, Стратегия — `CardStrategyView.vue`).
Общие переиспользуемые: `MoneyInput.vue` (ввод суммы с валютой и живыми эквивалентами,
`v-model` по `{ amount, currency }`) и `ScheduleEditor.vue` (`v-model` по schedule).
Компоненты читают глобальный `state` напрямую и мутируют его через CRUD-хелперы стора —
props/эмиты не пробрасываются сверху. Стили — один глобальный `src/styles.css` с
CSS-переменными (`--accent`, `--panel`, `--border`, `--muted`) плюс scoped-стили в SFC.

## Соглашения

- Комментарии и весь UI-текст на русском. Прямые кавычки `"..."`, без длинных/средних тире
  (`—`/`–`) в НОВОМ коде и комментариях — дефис `-`. Старые тире в коде с ранних коммитов не
  трогаем без нужды (в UI `'—'` часто осознанный placeholder).
- Новая логика — чистой функцией на правильном слое + тест: базовый расчёт в `finance.js`
  (`test/finance.test.js`), ходы/сценарии/переносы в `scenarios.js` (`test/scenarios.test.js`),
  рекомендации в `advice.js` (`test/advice.test.js`). Не внутри компонента — компоненты
  тонкие (читают state, зовут чистые функции, мутируют через CRUD/прямой доступ).
- Любую денежную величину держи как `{ amount, currency }` и конвертируй в рубли только в
  точке расчёта. Разные валюты не складывай напрямую.
- Ставки (`apr`) — доля (0.619 = 61.9%). Проценты за период: `apr × сумма × дни/365`.
