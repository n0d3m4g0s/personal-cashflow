# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## О проекте

Семейный финансовый менеджер (Vue 3 + Vite, статическая сборка без бэкенда). Считает
денежный поток по дням, обслуживает много кредитных карт с разными датами выписок и
льготными периодами, кредиты, доходы/расходы с гибкой регулярностью, финансовые цели с
ETA-прогнозом. Мультивалюта (RUB базовая, AMD, USD). Данные живут в localStorage,
переносятся через экспорт/импорт JSON. Интерфейс полностью на русском.

## Команды

```bash
npm install
npm run dev        # dev-сервер, http://localhost:5173
npm run build      # статика в dist/
npm run preview    # предпросмотр собранной статики
npm test           # тесты движка (node --test, чистый Node, без раннера)
```

Один тест-файл: `test/finance.test.js`. Запуск отдельного теста:
`node --test --test-name-pattern 'monthlyFactor'`. Тесты покрывают только чистые функции
из `src/finance.js` — Vue-компоненты не тестируются.

Деплой: пуш в `main` собирает и публикует на GitHub Pages (`.github/workflows/pages.yml`).
`vite.config.js` использует `base: './'` (относительные пути ассетов), чтобы одинаково
работало и на Pages в подкаталоге, и при локальном preview.

## Архитектура

Три слоя: **чистое финансовое ядро** (тестируемое) → **реактивный стор** (localStorage) →
**Vue-компоненты по вкладкам**.

### Ядро (чистые функции, без Vue)

- `src/money.js` — мультивалюта. Каждая денежная величина хранится как
  `{ amount, currency }` в **нативной** валюте записи. `rates` = сколько единиц валюты в
  1 рубле (`amdPerRub`, `usdPerRub`). Всё сводится к рублю через `moneyToRub()` перед
  арифметикой. Никогда не складывай суммы в разных валютах напрямую — сначала в рубли.
- `src/finance.js` — даты, регулярность, движок прогноза, цели. **Даты локальные без
  сдвига TZ** (`parseDate`/`fmtISO`/`addMonths` работают через `new Date(y, m, d)`, а не
  через UTC-парсинг ISO-строк — это осознанно, не «чини»).

### Ключевые концепции движка

- **Schedule** (`{ frequency, interval, customUnit, startDate, endDate }`) — регулярность
  записи. `expandSchedule()` разворачивает её в конкретные даты внутри диапазона;
  `monthlyFactor()` даёт множитель «сколько раз в месяц» для месячных эквивалентов.
  Частоты: `once | weekly | biweekly | monthly | quarterly | yearly | custom`.
- **buildForecast(state, opts)** — главный движок. Собирает события (доходы +, расходы −,
  кредиты −, карты −), сортирует по дате, считает нарастающий остаток от `startingCash`,
  формирует `alerts` при падении ниже `safetyBuffer`. Возвращает `{ days, events, alerts,
  minBalance, endBalance, ... }`. Кредитки дают **одно** ближайшее обязательство на карту
  (снимок текущего долга), а не рекуррентный ряд; `payStrategy: 'full'` гасит выписку
  целиком (льготный период, без процентов), `'minimum'` — только минимальный платёж.
- **buildMonthly()** — усреднённая картина «в среднем за месяц» (доход/расход/обязательства/
  профицит), используется дашбордом и целями.
- **computeGoals(state, opts)** — ETA по целям. Свободный месячный профицит распределяется
  **последовательно по приоритету** на цели без своего взноса; цели с ручным
  `monthlyContribution` считаются параллельно. `opts.extraIncome/extraExpense` — режим
  «играем цифрами» (what-if).

### Стор

`src/store.js` — `reactive(load())`, автосохранение в localStorage (debounce 200 мс,
ключ `family-finance:v1`). `migrate()` добивает отсутствующие поля при загрузке старых
данных — при добавлении новых обязательных полей в модель обновляй `migrate()`.
CRUD через `addItem/removeItem/duplicateItem(kind, ...)`, где `kind` ∈
`income | expense | loan | card | goal`. `src/seed.js` (`makeSeed()`) — стартовый префилл;
поле `note` на записях — подсказка-плейсхолдер для пользователя, не логика.

### UI

`src/App.vue` — вкладки (dashboard/forecast/goals/cards/loans/income/expenses/settings),
переключение через `<component :is>`. Каждая вкладка — компонент в `src/components/`.
Общие переиспользуемые: `MoneyInput.vue` (ввод суммы с валютой и живыми эквивалентами,
`v-model` по `{ amount, currency }`) и `ScheduleEditor.vue` (`v-model` по schedule).
Компоненты читают глобальный `state` напрямую и мутируют его через CRUD-хелперы стора —
props/эмиты не пробрасываются сверху. Стили — один глобальный `src/styles.css` с
CSS-переменными (`--accent`, `--panel`, `--border`, `--muted`) плюс scoped-стили в SFC.

## Соглашения

- Комментарии и весь UI-текст на русском.
- Новая финансовая логика — чистой функцией в `src/finance.js` (или `money.js`) + тест в
  `test/finance.test.js`, а не внутри компонента. Компоненты остаются тонкими.
- Любую денежную величину держи как `{ amount, currency }` и конвертируй в рубли только в
  точке расчёта.
