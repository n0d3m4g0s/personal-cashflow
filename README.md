# 🐾 Семейные финансы (муж · жена · 2 мопса)

Приложение для семейного финансового менеджмента: много кредитных карт с разными
датами выписок и льготными периодами, ипотека и кредит, аренда в Ереване, расходы
ИП в Армении, мультивалютность и **финансовые цели с прогнозом даты закрытия**.

Отвечает на главные вопросы:

- **Хватает ли денег** — сводка доход vs обязательные расходы, профицит/дефицит.
- **Когда и куда вносить платежи** — календарь-прогноз денежного потока по дням.
- **Сколько накопить к какому числу / где занять** — алерты по кассовым разрывам:
  в какие даты остаток опустится ниже безопасного буфера и на сколько.
- **Когда закроется цель** — ETA по каждой цели + режим «играем цифрами».

## Технологии

- **Vue 3 + Vite**, статическая сборка (без бэкенда).
- Данные хранятся в браузере (**localStorage**). Экспорт/импорт JSON для бэкапа
  и переноса между устройствами — данные никуда не отправляются.
- Мультивалюта: рубль (базовая) + драм (AMD) + доллар (USD), курсы редактируются,
  ввод в любой валюте пересчитывает эквиваленты.

## Запуск локально

```bash
npm install
npm run dev       # http://localhost:5173
```

Сборка статики и предпросмотр:

```bash
npm run build     # результат в dist/
npm run preview
```

Тесты движка прогноза:

```bash
npm test
```

## Деплой на GitHub Pages

В репозитории настроен workflow `.github/workflows/pages.yml`: при пуше в `main`
проект собирается и публикуется на GitHub Pages.

Один раз включите Pages: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
После этого сайт будет доступен по адресу `https://<user>.github.io/stuff/`.

`vite.config.js` использует относительный `base: './'`, поэтому сборка одинаково
работает и на Pages в подкаталоге, и при локальном `preview`.

## Как пользоваться

1. **Настройки** — укажите текущий остаток денег, безопасный буфер, горизонт
   прогноза и курсы валют.
2. **Кредитки** — по каждой карте: день выписки, день платежа, льготный период,
   минимальный платёж, текущий долг и сумму выписки. Стратегия «полное погашение»
   держит льготный период (без процентов), «минимум» — показывает тугой сценарий.
3. **Кредиты** — ипотека 117 000 и потребкредит 27 000 (укажите остаток долга,
   чтобы прогноз знал дату закрытия).
4. **Доходы / Расходы** — суммы и **гибкая регулярность** (разово, еженедельно,
   раз в 2 недели, ежемесячно, квартал, год или свой интервал) с датой начала и
   окончания. Всё это учитывается в прогнозе.
5. **Цели** — прогресс, дата закрытия, дедлайны-светофоры и «играем цифрами»
   (двигайте гипотетические доход/расходы и смотрите новые даты).
6. **Дашборд** — сводка месяца, ближайшие платежи, алерты.
7. **Прогноз** — таймлайн по дням с нарастающим остатком; красным подсвечены дни
   ниже буфера.

## Модель данных

Всё описано в `src/` — чистая логика в `src/finance.js` (даты, регулярность,
прогноз, цели) и `src/money.js` (мультивалюта). Стартовые данные (префилл) —
`src/seed.js`; правьте/удаляйте под себя прямо в интерфейсе.
