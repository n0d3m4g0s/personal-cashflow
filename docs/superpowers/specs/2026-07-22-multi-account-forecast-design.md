# Раздельные счета в прогнозе денежного потока

Дата: 2026-07-22

## Проблема

Сейчас весь прогноз считает один общий остаток от единственного `settings.startingCash`.
Много расходов идёт в долларах, но просадку именно на долларовом счёте не видно: пока
общий остаток (сведённый в рубли) положительный, кассового разрыва не показывается, даже
если долларовый счёт ушёл в минус.

Нужно: раздельные счета с собственным стартовым капиталом, прогноз общего остатка ПЛЮС
раздельные просадки по каждому счёту, и алерты на уровне счёта.

## Решения (зафиксированы на брейнсторме)

1. **Один счёт = одна валюта.** У счёта фиксированная валюта (RUB/AMD/USD). Остаток и
   раздельная просадка считаются в валюте счёта. Курсы нужны только чтобы свести общий итог
   в рубли.
2. **Валюта записи независима от валюты счёта.** Доход/расход/кредит может быть в любой
   валюте; при расчёте баланса конкретного счёта сумма конвертируется в валюту счёта по
   курсу (заплатил за USD-подписку с рублёвого счёта - на рублёвой дорожке она в рублях).
3. **Привязываем к счетам: доходы, расходы, кредиты.** Кредитки (card) НЕ трогаем - это
   отдельный кредитный контур; погашение карты остаётся событием прогноза, но к счёту на
   этом этапе не привязывается.
4. **Миграция: всё на "Основной" счёт.** Один RUB-счёт "Основной" с текущим startingCash,
   к нему привязываются все старые записи автоматически. Разнесение по валютным счетам -
   вручную после миграции.
5. **UI прогноза: общий остаток + переключатель по счетам.** Сверху общий остаток
   (сведённый в RUB), переключатель "Все счета / <счёт>"; при выборе счёта таблица
   показывает только его движения и остаток в его валюте.
6. **Алерты: посчётный буфер.** У каждого счёта своё поле буфера (в валюте счёта). Алерт
   срабатывает, когда остаток счёта падает ниже его буфера. Общего буфера в настройках для
   расчёта больше нет.

## Архитектурный подход

Счета - первоклассная сущность в чистом ядре (`finance.js`), прогноз считает per-account.
Ложится на существующую слоистость: логика в тестируемом ядре, Vue-компоненты остаются
тонкими (читают state, зовут чистые функции, мутируют через CRUD).

Направление зависимостей не меняется: `advice.js -> scenarios.js -> finance.js -> money.js`.

## Модель данных

### Новая сущность `account` (`state.accounts[]`)

```js
{
  id: 'acc_...',
  name: 'Основной',        // произвольное имя
  currency: 'RUB',         // RUB | AMD | USD - валюта счёта, одна
  startingBalance: 25000,  // ЧИСЛО в валюте счёта (стартовый остаток на дату прогноза)
  safetyBuffer: 50000,     // ЧИСЛО в валюте счёта (посчётный буфер; 0 = следим только за минусом)
  note: '',
  disabled: false
}
```

`startingBalance` и `safetyBuffer` - числа в валюте счёта (НЕ `{amount,currency}`-объекты),
т.к. валюта задана самим счётом. Упрощает форму и расчёт.

### Записи получают `accountId`

- `income`, `expense`, `loan` - новое поле `accountId` (ссылка на `account.id`, либо `null`).
- `card` - без изменений.
- Валюта записи (`amount`/`currency`) остаётся независимой.

### Настройки

`settings.startingCash` и `settings.safetyBuffer` больше не читаются прогнозом (источник
истины - `accounts[]`). Оставляем поля в объекте ради обратной совместимости импорта.

## Движок прогноза (`buildForecast`)

Каждое событие получает `accountId` (из породившей записи; для карт `null`).

Два уровня расчёта:

1. **Общий баланс (в RUB) - как сейчас.** Старт = сумма `startingBalance` всех счетов,
   сведённых в RUB (число × курс). События в рублях, нарастающий остаток. Поля
   `days/endBalance/minBalance/minBalanceDate` сохраняются (обратная совместимость).

2. **Per-account дорожки - новый `perAccount[]`.** Для каждого счёта:
   - старт = `account.startingBalance` (в валюте счёта);
   - события этого счёта конвертируются в валюту счёта (нативная сумma события -> валюта
     счёта через `convert`);
   - нарастающий баланс в валюте счёта, свой `days[]`, `minBalance`, `minBalanceDate`,
     `endBalance`;
   - посчётные алерты: остаток < `account.safetyBuffer`; отдельный флаг `belowZero` (< 0).

### Форма возврата

```js
buildForecast(state, opts) -> {
  // общий уровень (поля сохраняются):
  start, end, events, days, alerts, minBalance, minBalanceDate, endBalance, buffer,
  // новое:
  perAccount: [
    { account, currency, startingBalance, days, alerts,
      minBalance, minBalanceDate, endBalance, buffer }
  ]
}
```

### Алерты

Общего буфера нет. Общий `alerts` = объединение посчётных алертов, каждый помечен
`accountId`/именем счёта, поле `shortfall` в валюте счёта. Так блок "Кассовые разрывы"
показывает "минус на долларовом счёте", даже когда суммарно плюс.

Записи без `accountId` (карты; несмигрированное) попадают в общий баланс, но не в
`perAccount`. В UI - псевдо-строка "Общий контур / без счёта", чтобы деньги не терялись.

`buildMonthly` / `computeGoals` остаются на общем уровне (в RUB) - про усреднённую картину,
разбивка по счетам им не нужна. Не трогаем.

## Миграция и стор

**`migrate()` (store.js)** - идемпотентный блок:
- Если `accounts` пуст/нет: создать счёт `{ name: 'Основной', currency: 'RUB',
  startingBalance: <settings.startingCash сведён в RUB>, safetyBuffer: <settings.safetyBuffer
  в RUB>, note: '' }`.
- Всем `income/expense/loan` без `accountId` проставить id "Основного".
- `settings.startingCash/safetyBuffer` оставить (совместимость), прогноз их не читает.

**CRUD** - добавить `account` в `collectionFor` (`addItem/removeItem/duplicateItem`). При
удалении счёта записи с этим `accountId` -> `accountId = null` (не каскадное удаление).

**`seed.js`** - префилл получает счёт "Основной" (RUB) и проставляет `accountId` записям.

**`clearAll()`** - `accounts: []` (migrate создаст "Основной" при чтении).

## UI

- **Новая вкладка "Счета"** (`AccountsView.vue`) - CRUD-таблица: имя, валюта, стартовый
  остаток, буфер, заметка. По образцу `IncomeView.vue`. Регистрируется в `App.vue` nav.
- **Селектор счёта** в `IncomeView / ExpensesView / LoansView` - поле "Счёт" (dropdown из
  `state.accounts`) рядом с "Владелец". Валюта записи - отдельное поле (независима).
- **`ForecastView.vue`** - переключатель "Все счета / <счёт>". "Все" = общий вид (RUB).
  Выбран счёт = движения и нарастающий остаток в валюте счёта из `perAccount`.
- **`Dashboard.vue`** - ряд карточек по счетам (текущий остаток, минимум на горизонте, дата
  просадки - в своей валюте). Блок "Кассовые разрывы" показывает посчётные алерты с именем
  счёта.
- **`SettingsView.vue`** - убрать "Текущий остаток" и "Буфер" из параметров прогноза
  (переехали в счета). Курсы валют и горизонт остаются.

## Тесты (`test/finance.test.js`)

- Два счёта в разных валютах -> `perAccount` даёт правильные раздельные остатки в своих
  валютах.
- Расход в USD, списанный с RUB-счёта -> конвертация в рубли на дорожке счёта.
- Посчётный алерт: долларовый счёт в минусе при плюсовом общем -> есть алерт с его accountId.
- Миграция без accountId -> всё падает на "Основной", общий баланс не изменился (регрессия).

## Вне scope (YAGNI)

- Переводы между счетами, история операций, банковский учёт.
- Привязка кредиток к счетам.
- Мультивалютный (несколько валют на одном) счёт.
- Разбивка `buildMonthly`/целей по счетам.
